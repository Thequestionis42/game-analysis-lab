# 生成 HTML 前的强制审计清单

## 一、产品理解审计

- [ ] 是否在正式分析前读取 `examples/GO_GO_MUFFIN_PRODUCT_UNDERSTANDING_EXAMPLE.md` 作为质量校准范例？
- [ ] 是否在方法口径中说明范例只用于校准分析结构和证据口径，未直接套用范例产品结论？
- [ ] 是否先回答“这是一款怎样的游戏”？
- [ ] 是否解释表层包装、核心品类、中层结构、战斗形态、长线驱动力和差异点？
- [ ] 是否输出分钟级、日级、周级和长周期循环？
- [ ] 是否明确主线体验本质？
- [ ] 是否识别主要战斗验证场？
- [ ] 是否提出前置研究假设并在后续验证？

## 二、主线审计

- [ ] 是否检索主线、关卡、章节、地图、区域、大陆、剧情、世界、转职、转生、英雄之路、英雄诗篇、新难度、通关主线？
- [ ] 是否遗漏明确写出“新增主线”的公告？
- [ ] 是否把秘境、绝境、Boss、团本或活动误算成严格主线扩容？
- [ ] 是否保留公告日期和实际生效日期？
- [ ] 是否优先使用实际生效日期计算间隔？
- [ ] 是否说明章节号跳跃和资料缺口？
- [ ] 图表数字是否能由主线明细复算？

## 三、阶段版本包审计

- [ ] 是否与严格主线扩容拆开？
- [ ] 是否固定使用版本包体量矩阵？
- [ ] 是否避免输出不可复算的单一模块分数？
- [ ] 是否分析主线空窗期由什么内容承接？

## 四、养成审计

- [ ] 是否检索装备、宠物、坐骑、配件、收藏、宝石、技能、核心、神器、神铸、融合、觉醒、突破、词条、继承、置换、迁移、回收？
- [ ] 每个系统是否只有一个主分类？
- [ ] 是否使用次标签表达跨系统关系？
- [ ] 是否误把 QoL 归类为新养成线？
- [ ] 是否误把内容扩充归类为系统深化？
- [ ] 是否识别代际扩展和资源迁移？
- [ ] 是否输出“系统生命周期演进分期图”？
- [ ] 分期图横轴是否按产品生命周期阶段读取，而不是机械罗列年份？
- [ ] 分期图纵轴是否覆盖关键系统线，并说明各阶段新增/强化内容、分类、玩家行为变化和产品作用？
- [ ] 分期图是否保留 HTML 可读文本，而不是只输出不可复核图片？

## 五、中后期机制审计

- [ ] 是否检索赛季、排行、跨服、公会、联盟、旅团、好友、好感、交换、交易、拍卖、家园、UGC、回流、追赶、互通、合服、托管、自动、一键、找回、商店、凭证、礼包、保底、自选？
- [ ] 是否覆盖主线、养成、活动、赛季化、竞争、社交、UGC、生命周期治理和商业化辅助？
- [ ] 是否区分已发现、未发现和资料不足？
- [ ] 是否避免将“未发现”写成“不存在”？

## 六、证据审计

- [ ] 每个关键结论是否至少有一个官方来源？
- [ ] 是否区分 A、B、C、D 四级证据？
- [ ] 攻略站是否仅用于解释机制？
- [ ] 玩家讨论是否未进入核心结论？
- [ ] 推断是否使用克制措辞？

## 七、边界审计

- [ ] 是否只分析一款产品？
- [ ] 是否删除产品 B 和跨产品比较章节？
- [ ] 是否未输出开发成本估算？
- [ ] 是否未输出玩家反馈专题？
- [ ] 是否未输出具体排期建议？
- [ ] 商业化是否只作为辅助轴？

## 八、HTML 交付审计

- [ ] 是否在生成 HTML 前读取 `examples/HTML_REPORT_STRUCTURE_EXAMPLE.md`？
- [ ] 是否包含正文 15 个章节？
- [ ] 是否包含附录 A-K？
- [ ] 正文章节顺序是否与结构范例和 `templates/REPORT_OUTLINE.md` 一致？
- [ ] Executive Summary 是否是结论卡，而不是目录复述？
- [ ] 是否有快速目录？
- [ ] 是否有可点击来源链接？
- [ ] 长表是否支持滚动？
- [ ] 图表是否可以从附录复算？
- [ ] 移动端是否可阅读？
